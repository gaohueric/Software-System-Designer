## 二、计算机体系结构与 CPU（核心得分点，⭐⭐⭐⭐⭐）

### 2.1 冯·诺依曼结构（⭐⭐⭐⭐）

- **核心关键词**：**存储程序原理、统一存储、顺序执行、单总线、冯·诺依曼瓶颈**
- **五大组成部件**：
  - **运算器（ALU）**：执行算术运算与逻辑运算。
  - **控制器（CU）**：负责"取指 → 译码 → 执行 → 取下一条指令"的控制流程。
  - **存储器（Memory）**：统一存储程序和数据。
  - **输入设备**：键盘、鼠标等。
  - **输出设备**：显示器、打印机等。

- **三大核心原理（必考概念）**：
  - **存储程序原理**：**程序和数据以二进制形式存放在同一存储器中**。
  - **程序控制原理**：计算机按程序自动执行，不需人工干预。
  - **顺序执行原理**：指令按地址顺序执行，只有遇到转移/跳转指令才改变流程。

- **冯·诺依曼瓶颈**：
  - **程序与数据共用一条总线** → 取指与取数**不能并行** → 形成性能瓶颈。

- **简易 ASCII 结构示意**（理解结构）：
  ```
  输入设备 ─→┐
             │
          ┌──▼──────┐
          │  存储器 │  ← 程序 + 数据（统一存储）
          └──┬──────┘
             │
        ┌────▼─────┐
        │   CPU    │ = 控制器 + 运算器 + 寄存器
        └────┬─────┘
             │
        输出设备
  ```

---

### 2.2 哈佛结构与 DSP（⭐⭐⭐⭐⭐）

- **哈佛结构核心特征**：
  - **程序存储器**与**数据存储器**物理分离。
  - **程序总线**与**数据总线**分离，可**同时取指令和取数据** → 高效率。
  - 结构相对复杂，成本高。

- **冯·诺依曼 vs 哈佛结构对比表（必背）**：

| 对比维度   | 冯·诺依曼结构                                 | 哈佛结构                                        |
|------------|-----------------------------------------------|-------------------------------------------------|
| 存储器     | **程序和数据共用一个存储器**                  | **程序存储器与数据存储器分离**                  |
| 总线       | 单一总线（指令/数据分时共享）                 | 程序总线 + 数据总线（可并行访问）               |
| 取指/取数  | 不能同时进行，形成**冯·诺依曼瓶颈**           | 可同时取指与取数，**吞吐率更高**                |
| 数据宽度   | 程序、数据宽度必须一致                        | 程序、数据宽度可以不同，更灵活                  |
| 硬件复杂度 | 结构简单，成本低                              | 结构复杂，成本高                                |
| 应用场景   | 通用计算机（PC、服务器）                      | **DSP、嵌入式系统、部分单片机**等               |

- **DSP（Digital Signal Processor，数字信号处理器）要点**：
  - **用途**：音频处理、图像处理、通信、雷达等**实时信号处理**。
  - **结构**：**通常采用哈佛结构**（高频考点）。
  - 典型选择题："DSP 一般采用的结构是？" → **哈佛结构**。

---

### 2.3 CPU 组成与寄存器（⭐⭐⭐⭐）

- **CPU 基本组成**：
  - **CPU = 运算器 + 控制器 + 寄存器组**

- **常考寄存器功能表（必记）**：

| 寄存器名称          | 缩写   | 主要功能（出题常用描述）                              |
|---------------------|--------|-------------------------------------------------------|
| **程序计数器**      | **PC** | 存放**下一条要执行指令的地址**                        |
| **指令寄存器**      | **IR** | 存放**当前正在执行的指令**                            |
| **累加寄存器**      | **AC** | 存放运算的操作数和中间结果                            |
| **数据缓冲寄存器**  | **DR** | 暂存从内存读入的数据或准备写回内存的数据              |
| **程序状态字寄存器**| **PSW**| 存放运算结果状态（如进位标志、零标志、溢出标志等）   |

- **典型考法**：
  - 问"存放下一条指令地址的是？" → **PC**。
  - 问"存放当前执行指令的是？" → **IR**。
  - 问"哪一个不属于控制器组成？" → **AC**（属于运算器）。

## 三、数据表示与编码（数制 & 码制，⭐⭐⭐⭐）

### 3.1 数制及其转换（⭐⭐⭐）

- **四种常见进制表示方式**：
  - **二进制**：`1011B` 或 `0b1011`
  - **八进制**：`123O` 或 `0123`
  - **十进制**：`123D` 或直接 `123`
  - **十六进制**：`A1BH` 或 `0xA1B`

- **常用转换方法**：
  - **二进制 → 十进制**（按权展开）  
    `(101101)_2 = 1×2^5 + 0×2^4 + 1×2^3 + 1×2^2 + 0×2^1 + 1×2^0 = 45`
  - **十进制 → 二进制**：
    - 整数部分：**除 2 取余，逆序排列**。
    - 小数部分：**乘 2 取整，顺序排列**。
  - **二进制 ↔ 八/十六进制**：
    - 二进制 → 八进制：从小数点开始，**三位一组**。
    - 二进制 → 十六进制：从小数点开始，**四位一组**。

> 真题一般直接给出数，要求转换结果，多为**送分小题**。

---

### 3.2 原码、反码、补码、移码（⭐⭐⭐⭐）

#### 3.2.1 定义对比（必考表）

| 编码方式 | 定义要点                                                        | 典型特点/考点                         |
|----------|------------------------------------------------------------------|----------------------------------------|
| **原码** | 最高位为符号位（0 表示正数，1 表示负数）；其余为数值绝对值     | **零有两种表示（+0/-0）**              |
| **反码** | 正数：反码=原码；负数：符号位不变，其余位取反                  | 仍然有正负零，运算不够方便             |
| **补码** | 正数：补码=原码；负数：补码=反码+1                              | **零的表示唯一**、加减运算可统一处理   |
| **移码** | 在真值的基础上加上一个固定偏移量（bias）                        | **常用来表示浮点数的阶码**             |

- **常见考点**：
  - "哪种编码使**零的表示唯一**并且便于运算？" → **补码**。
  - "浮点数阶码一般采用哪种形式表示？" → **移码**。

---

### 3.3 定点数与浮点数（⭐〜⭐⭐）

- **定点数**：
  - 小数点的位置**固定**。
  - 表示范围较小，适用于简单场景。
- **浮点数**：
  - 小数点的位置**可变**，通过"阶码 + 尾数"表示。
  - 一般格式：**数符 + 阶符 + 阶码 + 尾数**。
  - 阶码通常采用**移码**，便于比较大小。

> 本部分通常作为背景知识出现，与性能/数值范围题偶尔结合考查。

## 四、校验码与数据可靠性（检错/纠错，⭐⭐⭐⭐⭐）

### 4.1 奇偶校验码（⭐⭐⭐）

- **原理**：在数据前/后增加 1 位**校验位**，使整个码字中"1"的个数满足：
  - **奇校验**："1"的总数为奇数。
  - **偶校验**："1"的总数为偶数。
- **能力**：
  - 能**检错**（发现奇数位错误）。
  - 不能**纠错**。
- **应用**：常用于**简单的数据传输场景**。

---

### 4.2 CRC（循环冗余校验码）（⭐⭐⭐⭐）

- **原理**：基于**多项式除法**，由被校验数据和生成多项式决定校验码。
- **能力**：
  - 能检出多种常见错误：单错、双错、突发错误等。
  - **只能检错，不能纠错**。
- **典型应用场景**：
  - **网络传输、数据链路层协议**（如以太网帧尾部的 CRC 校验）。

> 真题更重视："**CRC 属于检错码/不能纠错，用在网络传输**"。

---

### 4.3 海明码（Hamming Code）（⭐⭐⭐⭐⭐）

- **核心能力**：
  - 能**发现单错、双错**。
  - 能**纠正单个错误（单错纠错）**。
- **关键公式（必背）**：
  `2^r ≥ m + r + 1`
  - m：数据位数
  - r：校验位数

- **典型题型**：
  - "要对 m 位数据进行海明校验，至少需要多少位校验位？" → 代入上式，从 r=1 开始试。

- **应用场景**：
  - 常用于**内存、存储系统**，提高数据可靠性。

---

### 4.4 三种校验码综合对比表（高频）

| 校验码   | 检错能力                                  | 纠错能力          | 典型应用场景             |
|----------|-------------------------------------------|-------------------|--------------------------|
| **奇偶校验** | 能发现奇数位错误                           | 无                | 简单串行通信、低成本场景 |
| **CRC**  | 能检出多种错误（单错、双错、突发错等）     | 无                | **网络通信、链路层**     |
| **海明码** | 能发现单错、双错                           | **能纠正单错**    | **内存、存储系统**       |

---

## 五、计算机性能指标（⭐⭐）

### 5.1 基本概念

- **字长**：CPU 一次能处理的二进制位数（如 32 位、64 位）。
- **主频（时钟频率）**：每秒钟时钟脉冲个数，主频越高，理论上处理能力越强。
- **存储容量**：内存及外存的存储空间大小。
- **存取周期**：存储器完成一次读/写操作所需时间。

### 5.2 常用性能公式（识别 + 简单代入）

- **MIPS**（每秒百万条指令）  
  `MIPS = 指令数 / (执行时间 × 10^6)`
- **CPI**（每条指令平均时钟周期数）  
  `CPI = 总时钟周期数 / 指令条数`
- **FLOPS**（每秒浮点运算次数）：体现**浮点运算能力**，多用于衡量高性能计算机。

## 六、第一小时核心记忆清单（可当小抄）

- **结构类**：
  - **冯·诺依曼结构**：**存储程序、统一存储、单总线、有瓶颈**。
  - **哈佛结构**：**程序数据分离、双总线、并行高效、硬件复杂**。
  - **DSP**：**采用哈佛结构**。

- **CPU&寄存器**：
  - **CPU = 运算器 + 控制器 + 寄存器组**。
  - **PC**：下一条指令地址；**IR**：当前指令；**AC**：运算中间结果。

- **数据表示**：
  - **补码**：零唯一、便于运算。
  - **移码**：用于表示**浮点数的阶码**。

- **校验码**：
  - **奇偶校验**：简单检错，不能纠错。
  - **CRC**：强检错能力，用于**网络/链路层**。
  - **海明码**：`2^r ≥ m+r+1`，**单错可纠，双错可检**，用在**内存**。

- **性能**：
  - 认识 **字长、主频、MIPS、CPI、FLOPS**，会代入简单公式即可。
